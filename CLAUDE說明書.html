一、專案是什麼

一個個人 AI 學習入口網站：由 Cowork 排程自動收集 AI 資訊，AI 預先篩選、評分、產出繁中摘要，你只需閱讀「值得看」的精華，解決 AI 資訊爆炸與學習焦慮。

核心原則

  1. AI 先篩選，人再閱讀——每日精選限量 5 則，不做資訊瀑布
  2. 繁體中文為主——英文內容附中文摘要，專有名詞保留英文
  3. 持續自動更新——每日新聞、每週工具/文章/知識庫
  4. 從學習出發——收藏/已讀/筆記推著你學，不只是看

完整規劃見《AI學習網站建置規劃書》（v1.1 定稿，在 AI工具 資料夾）。

二、建置經過（決策紀錄）

  1. 需求訪談：提出 5 大需求——每日 AI 新聞、AI 工具庫（依應用分類+開源精選）、AI 知識庫（自動更新）、生產力工具區、大神文章區。
  2. 規劃書：Claude 撰寫建置規劃書，補充第 6 模組「我的學習」，提醒 FB 無法自動抓取、資訊過載風險、版權界線等。
  3. 方案選擇：比較 4 個部署方案後，選定方案 D：Cowork 排程 + GitHub Pages——參考你已成功運作的 iPAS 備考網站模式。$0 成本、AI 處理由 Cowork 完成、手機可看。
  4. 定案決策：遊戲化不採用；每日精選 5 則；Phase 3 做 Email/LINE 推播；興趣領域＝生產力工具最高權重。
  5. 建置：Claude 寫出網站（純 HTML/JS/CSS 無框架）、初始內容、真實新聞首批 5 則，設定 2 個排程任務。
  6. 部署：Claude 用 Chrome 自動化：建 GitHub repo → 上傳檔案 → 開啟 GitHub Pages → 驗證上線成功。

三、技術架構

使用者瀏覽 ──→ GitHub Pages（靜態網站，手機可看） ↑ 資料檔上傳（自動覆蓋同名檔） Cowork 排程任務 ──→ WebSearch 抓資訊 → AI 篩選/摘要 → 更新本機 data/*.js → Chrome 自動上傳 GitHub → 網站更新
  • 無後端、無資料庫、無 API 費用——網站是純靜態檔案，資料存在 data/*.js
  • 個人紀錄（收藏/已讀/筆記）存在瀏覽器 localStorage，不上傳雲端
  • 成本 $0（含在 Claude 訂閱內）

檔案結構

ai-learning-hub/ ├── CLAUDE.md # 專案說明書（本文件的 md 版） ├── README.md # 部署與更新簡要說明 ├── index.html # 網站主頁（六大模組 + UI 樣式） ├── app.js # 前端邏輯（分頁、收藏/已讀、深淺色、AI導師） └── data/ # 資料檔（排程更新的對象） ├── news.js # 每日新聞 window.SITE_NEWS ├── tools.js # 工具庫 window.SITE_TOOLS ├── knowledge.js # 知識庫 window.SITE_KNOWLEDGE ├── productivity.js # 生產力工具 window.SITE_PRODUCTIVITY └── articles.js # 大神文章 window.SITE_ARTICLES

資料格式重點

  • 每個 data 檔都是 window.SITE_XXX = {...} 的 JS 檔（不用 JSON 是為了本機開啟不受 CORS 限制）
  • news.js：lastUpdated + picks（5 則精選）+ archive（歷史最多 60 則）；每則含 date / stars / category / title / summary / why / source / url
  • tools.js：categories（9 大分類）+ weekly（每週精選）+ toolVideos（本週新增 YouTube 教學影片，每工具上限 5 支）+ toolVideosHistory（歷史累積，每工具上限 30 支）——兩個 key 都涵蓋全部工具名稱，做法比照 iPAS 網站
  • articles.js：leaders（追蹤名單：國際/中文/YouTube）+ articles（文章摘要）
  • 修改後務必 node --check 驗證語法

五、日常使用方式

  1. 看網站：開線上版（手機也行）或本機直接開 index.html（本機資料可能比線上新，因為排程先更新本機再上傳）
  2. 每天：看首頁「今日精選」5 則，有興趣按「☆ 收藏」，看完按「標記已讀」
  3. 每週：看工具庫「本週精選」與大神文章新摘要，挑 1-2 個實際動手玩
  4. AI 導師：知識庫/生產力頁每個條目都有「🤖 呼叫 AI 導師」按鈕，點了自動複製提問 Prompt，貼到 Claude 深入學＋拿 10 分鐘小練習
  5. 我的學習：收藏待學清單＋已讀紀錄＋學習筆記（存瀏覽器本機，勿用無痕模式，換裝置不同步）

你是非工程師，所有維護都透過 Cowork 對 Claude 下指令完成：

內容沒更新檢查：①電腦當時是否開機 ②Scheduled 任務執行紀錄 ③Chrome 是否登入 GitHub
手動更新一次到 Scheduled 按「Run now」，或直接叫 Claude 執行一次更新
網站改版/加功能直接描述需求，Claude 改 index.html / app.js 後上傳

手動上傳 GitHub（備援）

自動上傳失敗時：到 repo → 進 data 資料夾 → Add file → Upload files → 拖入更新後的檔案 → Commit changes（同名自動覆蓋），約 1 分鐘後網站生效。

資料備份

所有內容都是純文字檔，GitHub repo 本身就是雲端備份；本機資料夾是第二份。建議偶爾整個資料夾複製備份。

七、已知限制與風險

  1. Facebook 無法自動抓取（平台限制）——改追蹤創作者的部落格/Substack/YouTube；FB 好文可手動貼給 Claude 摘要後加入
  2. 電腦沒開機當天不更新——隔次開機補跑；要完全雲端化可升級 GitHub Actions（規劃書方案 B）
  3. AI 摘要可能有誤——重要內容以原文連結為準
  4. localStorage 限制——收藏/筆記在單一瀏覽器本機；清除瀏覽器資料會消失；線上版和本機版的紀錄是分開的
  5. 來源可能失效——排程總結會註明抓取失敗的來源，累積失效請叫 Claude 更換
